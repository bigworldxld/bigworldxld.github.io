广告业务

广告

一、广告的主要分类及应用场景

1. 按形式分类

  • 展示广告（Display Ads）
    • 应用：品牌曝光、产品推广（如横幅广告、插屏广告）。
    • 示例：网站侧边栏广告、APP开屏广告。
  • 视频广告（Video Ads）
    • 应用：故事化品牌传播（前贴片、信息流视频）。
    • 示例：YouTube前贴片广告、抖音短视频广告。
  • 原生广告（Native Ads）
    • 应用：融入内容提升用户体验（信息流、推荐列表）。
    • 示例：微博信息流广告、今日头条文章推荐。
  • 搜索广告（Search Ads）
    • 应用：精准关键词引流（竞价排名）。
    • 示例：Google Ads、百度竞价广告。
  • 社交广告（Social Media Ads）
    • 应用：互动营销（粉丝定向、KOL合作）。
    • 示例：Facebook动态广告、微信朋友圈广告。
  • 户外广告（OOH Ads）
    • 应用：本地化品牌曝光（交通枢纽、商圈）。
    • 示例：地铁灯箱、电梯分众传媒广告。

2. 按目的分类

  • 品牌广告：提升长期认知（如可口可乐的节日广告）。
  • 效果广告：直接转化（如电商促销广告）。
  • 公益广告：传播社会理念（如环保倡议）。

3. 按技术手段分类

  • 程序化广告：自动化投放（DSP平台实时竞价）。
  • 互动广告：用户参与（AR试妆、H5小游戏）。

二、市场主要广告供应商

1. 国际巨头

  • Google Ads：覆盖搜索、YouTube、展示网络。
  • Meta Ads（Facebook/Instagram）：社交广告主导者。
  • Amazon Advertising：电商场景内广告。
  • TikTok Ads：短视频年轻用户群体。

2. 中国本土供应商

  • 字节跳动（巨量引擎）：抖音、今日头条信息流广告。
  • 腾讯广告：微信、QQ等社交生态。
  • 百度营销：搜索+信息流组合。
  • 阿里巴巴（阿里妈妈）：淘宝/天猫电商广告。
  • 分众传媒：电梯/楼宇屏幕广告霸主。

3. 细分领域供应商

  • 蜂巢广告（Hive Box）
    • 定位：专注于社区场景的户外广告，覆盖快递柜、智能终端。
    • 应用：通过快递柜屏幕投放本地生活服务（如外卖、教育机构广告）。
    • 优势：高频触达社区人群，适合O2O行业。
  • 其他垂直类：
    • AdTiming（移动程序化广告）。
    • 汇量科技（Mobvista）（出海广告平台）。

三、新兴趋势与技术

  • AI驱动广告：动态生成创意（如ChatGPT写文案）。
  • 隐私合规工具：应对iOS隐私政策（如SKAdNetwork）。
  • 元宇宙广告：虚拟场景品牌植入（如耐克在Roblox建店）。

总结

广告形式多样，选择需结合目标（品牌/效果）、受众及预算。蜂巢广告等细分供应商适合特定场景（如社区），而巨头平台适合广泛覆盖。程序化购买和数据分析能力已成为行业标配

一、外卖广告业务的核心分类

1. 搜索广告（竞价排名）

  • 定位：商家通过竞价购买关键词排名，提升在外卖/大众点评APP内的搜索曝光。
  • 应用场景：
    • 用户搜索“火锅”“外卖”时，付费商家的店铺或商品优先展示。
    • 示例：某火锅店竞价“重庆火锅”关键词，排名靠前吸引点击。
  • 计费模式：CPC（按点击付费）或CPT（按时间付费）。

2. 信息流广告（原生展示）

  • 定位：在外卖APP首页、推荐页等位置插入原生广告，类似社交媒体信息流。
  • 应用场景：
    • 首页“猜你喜欢”板块插入餐饮、酒店推广；
    • 大众点评“探店笔记”中植入商家软广。
  • 优势：基于用户行为数据（如浏览、下单记录）精准推荐。

3. 商户通（店铺装修+推广）

  • 定位：一站式店铺运营工具，包含基础展示和付费推广功能。
  • 核心功能：
    • 店铺装修：优化主页、菜品图片、活动海报；
    • 推广通：付费提升店铺在列表页的排名（如“附近商家”Top 3）；
    • 团购推广：置顶团购套餐增加销量。
  • 收费模式：年费制（基础功能）+额外推广预算。

4. 到店广告（场景化曝光）

  • 定位：针对到店消费场景（餐饮、丽人、医美等）的垂直广告。
  • 形式：
    • 首页Banner：大促活动（如“517吃货节”）主会场入口；
    • 频道页焦点图：细分行业（如“美容”频道）的头部广告位；
    • 交易完成页广告：用户支付后推荐关联服务（如吃完火锅推KTV优惠券）。

5. 外卖广告

  • 定位：针对外卖商家的流量竞争工具。
  • 形式：
    • 列表页排名：竞价“黄金展位”（如第1、5、10等固定位）；
    • 满减标签：付费获取“限时特惠”“爆单推荐”等标签；
    • 红包推广：商家补贴用户红包，平台匹配流量（如“天天神券”活动）。

6. 联名营销（品牌合作）

  • 定位：与头部品牌联合打造定制化活动。
  • 案例：
    • 与可口可乐合作“可乐+汉堡”套餐，在APP开屏页推广；
    • 与电影《热辣滚烫》联动，推出“观影+餐饮”优惠券组合。

二、广告的技术与数据支撑

  1. 用户画像系统：
    • 基于地理位置、消费历史、搜索行为等数据定向投放（如向常点奶茶的用户推新茶饮店）。
  2. 实时竞价（RTB）：
    • 外卖广告位通过算法动态竞价，商家可设置每日预算上限。
  3. 效果归因工具：
    • 追踪广告点击到实际到店/下单的转化路径，优化ROI。

三、广告的收费模式

广告类型 主要计费方式 适用客户
搜索广告 CPC（点击付费） 中小商家
信息流广告 CPM（千次曝光付费） 品牌连锁店
商户通 年费+CPC附加推广 所有到店商家
外卖广告 CPC+固定位竞价 外卖高频品类（奶茶、快餐）
联名营销 固定费用+分成 头部品牌（如麦当劳、星巴克）

四、典型案例分析

  1. 案例1：某连锁奶茶店
    • 策略：购买“奶茶”关键词搜索广告+信息流曝光。
    • 结果：搜索排名提升30%，新客订单增长50%。
  2. 案例2：本地火锅店
    • 策略：开通“商户通”优化店铺页面，叠加“到店Banner”推广。
    • 结果：页面停留时间延长2倍，周末满座率提升。

五、竞争对比（外卖 vs 其他平台）

维度 外卖 抖音本地生活 支付宝口碑
优势 高频消费场景、闭环交易数据 流量爆发力强、内容种草 支付后场景渗透
劣势 广告成本逐年上升 转化链路较长（需跳转） 用户主动打开率低
适合商家 刚需品类（餐饮、外卖） 新奇特体验类（网红店） 连锁品牌会员运营

总结

外卖的广告业务本质是将流量精准分配给付费商家，核心优势在于：

  1. 场景闭环：从看到广告到下单、核销全流程在APP内完成；
  2. 数据深度：8亿用户吃喝玩乐行为形成精准投放基础；
  3. 产品矩阵：覆盖搜索、信息流、外卖、到店等多触点。

对商家的建议：中小商家可从“商户通+搜索广告”切入，品牌连锁需结合联名营销和全渠道曝光。

一、测试内容

1. 计费逻辑测试

  • 测试目标：验证计费规则是否正确执行（如CPC、CPM、CPA等）。

    CPA（Cost Per Action，每次行动成本） 是广告计费模式的一种，指广告主仅在用户完成 特定转化行为 时才支付费用。

    • 核心逻辑：按效果付费，避免无效曝光或点击的浪费。
    • 常见转化行为：
      • 表单提交（如注册、留资）
      • 应用下载
      • 购买下单
      • 电话咨询

CPA vs 其他广告指标对比

指标 计费触发条件 适用阶段 风险承担方
CPA 用户完成指定行为 转化阶段 媒体/渠道方
CPC 用户点击广告 流量引入阶段 广告主
CPM 广告展示1000次 品牌曝光阶段 广告主
CPS 用户实际支付订单 成交阶段 媒体/渠道方

关键区别：

  • CPA的“Action”不一定是购买（可能是留资），而 CPS（Cost Per Sale）必须达成销售。
  • CPA对广告主更安全，但对媒体技术要求高（需追踪深层转化）

技术实现要点

  1. 转化追踪：
    • 埋点技术：在转化页面（如支付成功页）部署SDK或像素代码。
    • 归因模型：确定用户转化归属（如最后一次点击/7天内首次曝光）。
  2. 反作弊机制：
    • 过滤虚假转化（如同一设备多次注册）。
    • 使用设备指纹、IP分析等技术识别异常行为。
  3. 数据同步：
    • 广告平台（如Facebook）、媒体、广告主的转化数据需实时对齐。

示例流程：
用户点击广告 → 跳转落地页 → 提交表单 → 触发CPA计费 → 数据回传广告平台。

优缺点分析

优点 缺点
广告主风险低，只为结果付费 媒体方承担流量质量压力
适合预算有限的中小企业 转化追踪技术复杂，成本高
易于计算ROI（如CPA<用户LTV） 竞争激烈时单价可能上涨

实际案例

案例1：在线教育机构

  • 目标：获取付费课程试听用户。
  • 策略：在抖音投放CPA广告，仅用户提交试听申请后付费（CPA=20元）。
  • 结果：500次转化，总成本1万元，后续试听→付费转化率30%，ROI达标。

案例2：电商APP

  • 目标：新用户首单转化。

  • 策略：与网盟合作，按CPA=50元结算（用户完成首单后付费）。

  • 风险控制：设置7天归因窗口，防止归因冲突

  • 测试内容：

    • 扣费准确性：广告点击/曝光后，是否按约定费率扣费。
    • 预算控制：日预算、总预算是否生效，超预算后是否停止投放。
    • 阶梯计价：如“消耗越多，单价越低”的梯度计费逻辑。

  • 示例：

    • 某广告主设置CPC（每次点击1元），测试100次点击是否扣费100元。
    • 若日预算设为50元，验证第51次点击是否停止计费。

ROI（Return on Investment，投资回报率）

定义：衡量广告或业务投入带来的收益效率，计算公式为：

ROI=(收益 - 成本)/成本×100%

核心意义：

  • 正ROI：盈利（如ROI=150%表示每投入1元赚回1.5元）。
  • 负ROI：亏损

LTV（Lifetime Value，用户终身价值）

定义：单个用户在其生命周期内为品牌贡献的总净收益。
​​计算逻辑​​（简化版）：

LTV=平均订单价值×购买频次×平均生命周期

核心意义：

  • 指导获客成本：若LTV=500元，则CPA（获客成本）应<500元。
  • 用户分层：高LTV用户需重点维护（如VIP会员）

应用场景：

  • 订阅制业务：计算用户续费带来的长期收益（如视频会员）。
  • 游戏行业：分析玩家付费深度（如氪金玩家的LTV）

（1）广告投放策略

  • 规则：CPA（单次获客成本）必须 < LTV，否则长期亏损。
  • 示例：
    • 教育机构LTV=5000元（学员总学费），则CPA可接受上限为2000元（ROI需>150%）。

（2）用户运营优化

  • 高LTV用户：通过个性化推荐、专属优惠提升复购（如亚马逊Prime会员）。
  • 低LTV用户：减少补贴，或引导至低成本渠道（如推送APP通知替代短信）。

（3）商业模式验证

  • 亏损期容忍度：若LTV远高于CPA，短期亏损可接受（如瑞幸咖啡早期扩张）

技术实现与数据分析

  1. LTV预测模型：
    • 使用机器学习（如生存分析、RFM模型）预测用户未来价值。
    • 工具：Python（Scikit-learn）、Google Analytics 4（预测性指标）。
  2. ROI实时监控：
    • 通过BI工具（如Tableau）追踪广告支出与收益的实时比率。

案例：

  • 某游戏公司用LTV模型发现，iOS用户LTV比安卓高30%，遂调整广告预算倾斜。

5. 行业案例

案例1：电商行业

  • 目标：大促活动ROI最大化。
  • 策略：
    • 预热期投放CPC广告（低成本引流）。
    • 爆发期聚焦CPS广告（ROI直接挂钩销售额）。
  • 结果：整体ROI达250%，高LTV用户复购占比40%。

案例2：SaaS企业

  • 挑战：免费试用用户转化率低。
  • 解决方案：
    • 计算试用用户的潜在LTV（基于行业均值）。
    • 针对高潜力用户定向推送付费功能引导。
  • 效果：试用→付费转化率提升20%，CPA降低15%

总结

  • ROI是短期效益标尺，回答“这次投入赚了吗？”
  • LTV是长期价值罗盘，回答“这个用户值得花多少钱获取？”
  • 关键动作：
    • 新业务初期优先验证LTV>CPA；
    • 成熟期通过分层运营提升LTV，优化ROI。

2. 结算对账测试

  • 测试目标：确保平台、广告主、渠道方的分账数据一致。
  • 测试内容：
    • 交易流水核对：订单金额、手续费、分成比例是否匹配。
    • 时间窗口对齐：T+1、T+7结算周期是否准确。
    • 异常场景：退款、部分结算、跨境汇率折算等。
  • 示例：
    • 广告主A消耗100元，平台分成30%，测试结算时是否打款70元给媒体方。

3. 数据一致性测试

  • 测试目标：验证前端展示、计费流水、结算报表的数据一致性。
  • 测试内容：
    • 埋点数据：曝光、点击等事件是否准确上报。
    • 数据聚合：小时级/天级汇总是否与原始日志一致。
    • 跨系统对齐：广告系统、财务系统、BI报表的数据是否同步。
  • 示例：
    • 前端显示广告曝光1000次，但计费系统日志仅记录800次，需排查丢失原因。

4. 异常场景测试

  • 测试目标：覆盖极端情况，防止资损。
  • 测试内容：
    • 高并发扣费：同一用户短时间内多次点击是否重复计费。
    • 恶意刷量：模拟机器人流量，验证反作弊规则是否生效。
    • 服务降级：计费服务宕机时，是否触发熔断机制。
  • 示例：
    • 某用户1秒内点击广告10次，系统应仅计费1次（防刷单）。

二、测试注意事项

  1. 数据隔离：
    • 测试环境需与生产环境隔离，避免误扣真实资金。
  2. 时间敏感性：
    • 结算测试需模拟真实时间窗口（如T+1），避免跨日数据错误。
  3. 幂等性设计：
    • 重复请求（如网络重试）不应导致多次扣费。
  4. 日志完整性：
    • 所有计费事件必须落库，支持事后审计。
  5. 合规性：
    • 涉及跨境结算时，需验证汇率计算和税务规则。

三、技术栈与核心知识点

1. 技术栈

技术领域 常用工具/框架 作用
数据埋点 Google Analytics、埋点SDK 采集用户行为数据（点击、曝光）
实时计算 Flink、Kafka 实时统计点击/曝光量，触发计费规则
批量对账 Hive、Spark SQL 离线核对订单与结算数据
数据库 MySQL（事务支持）、Redis（缓存） 存储计费流水、预算状态
监控报警 Prometheus、Grafana 监控扣费异常、延迟
反作弊 规则引擎（Drools）、机器学习模型 识别刷量、虚假流量

2. 核心知识点

  • 分布式事务：
    • 计费与结算可能跨多个服务，需保证数据一致性（如使用TCC、Saga模式）。
    • 示例：用户点击广告后，先冻结金额，确认有效后再扣费。
  • 幂等设计：
    • 通过唯一ID（如订单号）避免重复处理同一请求。
  • 数据一致性：
    • 采用CDC（Change Data Capture）同步数据库与数据仓库。
  • 财务合规：
    • 结算需符合会计准则（如收入确认规则IFRS 15）。

四、案例说明

案例：广告计费测试

  1. 测试场景：
    • 商家购买“黄金展位”（CPT计费，每天固定费用50元）。
  2. 测试步骤：
    • 正常流程：验证24小时内展示是否扣费50元。
    • 异常流程：
      • 模拟服务中断1小时，恢复后是否补扣费用。
      • 同一用户多次浏览，是否仅计费一次。
  3. 技术实现：
    • 用Flink实时统计曝光量，达到阈值后停止计费。
    • 通过MySQL事务保证预算扣减和流水记录的原子性。

五、总结

商增业务的计费结算测试需重点关注：

  1. 逻辑准确性：扣费、预算、分成规则必须与合同一致。
  2. 数据一致性：多系统间的数据对齐是核心难点。
  3. 异常容灾：高并发、服务故障等场景下不能资损。
  4. 技术兜底：通过分布式事务、幂等设计、反作弊等技术保障可靠性

实际建议：

  • 自动化测试覆盖80%常规场景（如Jmeter压测、脚本对账）
  • 灰度发布时逐步放量，监控核心指标（如扣费成功率）

其他

Revenue Per Click（每次点击收入）‌
‌广告主为有效点击支付的单价‌，计算公式为：总收入 ÷ 点击量。

‌应用场景‌：
‌效果广告优化‌：监测单次点击的收益贡献，结合ACoS（广告销售成本）评估投放盈亏（例：RPC为￥2，ACoS为40%，则每点击利润￥1.2）
‌竞价策略核心‌：亚马逊SP广告中，系统基于RPC动态调整广告位竞价（高RPC广告位可溢价竞价）

‌Remote Procedure Call（远程过程调用）‌
‌广告系统架构中的跨服务器通信协议‌，用于高性能数据传输。
‌技术特性‌：
‌低延迟传输‌：基于HTTP/2协议多路复用，支持万级QPS的广告请求分发（如阿里广告系统异步调用设计）
‌跨语言兼容‌：gRPC、Thrift等框架实现Java/Python等服务间调用（例：广告引擎调用反作弊服务）
‌核心价值‌：解决分布式系统中广告检索、计费、数据上报等模块的高效协同

RPT 边际转换率（广告效果指标）‌
‌定义‌：广告点击后实现最终转化的比例，即 ‌转化率（CVR）‌ 的另一种表述形式。
‌计算公式‌：
RPT边际转换率 = 最终转化次数 ÷ 点击次数
‌应用场景‌：

衡量广告投放效果的转化效率（例：点击100次产生5次购买，RPT边际转换率为5%）
用于优化电商广告的投放策略（如低RPT时需调整目标人群或创意）
