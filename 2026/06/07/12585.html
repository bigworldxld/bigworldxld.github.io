Claude Code交互⽅式：

  • 交互模式：运⾏ claude 启动 REPL 会话⼀次性模式：
  • 使⽤ claude -p “查询” 进⾏快速命令CLI 命令
# 启动交互模式
claude
# 以初始查询启动
claude "解释这个项⽬"
# 运⾏单个命令并退出
claude -p "这个函数做什么？"
# 处理管道内容
cat logs.txt | claude -p "分析这些错误"

claude -c 继续最近的对话 claude -c
claude -c -p “查询” 在打印模式下继续 claude -c -p “检查类型错误”
claude -r “<会话ID>” “查询” 通过 ID 恢复会话 claude -r “abc123” “完成这个 PR”
claude update 更新到最新版本 claude update

claude mcp 配置 Model Context Protocol 服务器

Vim 模式
Claude Code ⽀持⼀部分 Vim 键绑定，可以通过 /vim 启⽤或通过 /config 配置。
⽀持的部分包括：
模式切换： Esc （到 NORMAL）， i / I ， a / A ， o / O （到 INSERT）
导航： h / j / k / l ， w / e / b ， 0 / $ / ^ ， gg / G
编辑： x ， dw / de / db / dd / D ， cw / ce / cb / cc / C ， . （重复）

Claude Code提供两种继续之前对话的选项：
–continue ⾃动继续最近的对话
–resume 显⽰对话选择器继续最近的对话

继续最近的对话

这会⽴即恢复您最近的对话，⽆需任何提⽰。

在⾮交互模式下继续

claude –continue –print “继续我的任务”

将 –print 与 –continue ⼀起使⽤可以在⾮交互模式下恢复最近的对话，⾮常适合脚本或⾃动化。

显⽰对话选择器

claude –resume

这会显⽰⼀个交互式对话选择器，显⽰：

  • 对话开始时间
  • 初始提⽰或对话摘要
  • 消息数量

使⽤箭头键导航并按Enter选择对话。

⼯作原理：

  1. 对话存储：所有对话都会⾃动保存在本地，包含完整的消息历史
  2. 消息反序列化：恢复时，整个消息历史都会被还原以保持上下⽂
  3. ⼯具状态：之前对话中的⼯具使⽤和结果会被保留
  4. 上下⽂恢复：对话会在保留所有先前上下⽂的情况下继续

提⽰：

对话历史存储在您的本地机器上
使⽤ –continue 快速访问您最近的对话
当您需要选择特定的过去对话时使⽤ –resume

恢复时，您将看到整个对话历史记录，然后再继续

恢复的对话使⽤与原始对话相同的模型和配置开始

创建有效的CLAUDE.md⽂件
何时使⽤： 您想设置⼀个CLAUDE.md⽂件来存储重要的项⽬信息、约定和常⽤命令。
1 为您的代码库引导CLAUDE.md

使⽤⼯作树创建隔离的编码环境何时使⽤： 您需要同时处理多个任务，并在Claude Code实例之间完全隔离代码。

了解Git⼯作树
Git⼯作树允许您从同⼀存储库中检出多个分⽀到单独的⽬录。每个⼯作树都有⾃⼰的⼯作⽬录，⽂件是
隔离的，同时共享相同的Git历史。在官⽅Git⼯作树⽂档中了解更多。
12 创建新⼯作树

# 创建带有新分⽀的⼯作树
git worktree add ../project-feature-a -b feature-a
# 或使⽤现有分⽀创建⼯作树
git worktree add ../project-bugfix bugfix-123

这会创建⼀个包含存储库单独⼯作副本的新⽬录。

3 在每个⼯作树中运⾏Claude Code

# 导航到您的⼯作树
cd ../project-feature-a
# 在这个隔离环境中运⾏Claude Code
claude

在另⼀个终端中：

d ../project-bugfix
claude

管理您的⼯作树

# 列出所有⼯作树
git worktree list
# 完成后移除⼯作树
git worktree remove ../project-feature-

每个⼯作树都有⾃⼰独⽴的⽂件状态，⾮常适合并⾏Claude Code会话
在⼀个⼯作树中所做的更改不会影响其他⼯作树，防⽌Claude实例相互⼲扰
所有⼯作树共享相同的Git历史和远程连接
对于⻓时间运⾏的任务，您可以让Claude在⼀个⼯作树中⼯作，同时您在另⼀个⼯作树中继续开发
使⽤描述性⽬录名称，以便轻松识别每个⼯作树的任务
记得根据项⽬的设置在每个新⼯作树中初始化开发环境。根据您的技术栈，这可能包括：JavaScript项⽬：运⾏依赖安装（ npm install 、 yarn ）

Python项⽬：设置虚拟环境或使⽤包管理器安装其他语⾔：遵循项⽬的标准设置流程
