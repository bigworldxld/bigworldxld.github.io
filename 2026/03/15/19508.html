视频异常检测工具 v4.0.0

📦 简介

全维度视频视觉异常检测工具，支持单视频检测、对比检测、批量检测和Excel批量对比检测。

最新功能：✨ Excel批量对比检测 - 自动下载原片并批量对比转码视频

🎯 核心功能

1️⃣ 单视频检测

检测单个视频的质量异常（黑屏、绿屏、马赛克、卡顿等）

python3 detect.py video.mp4

2️⃣ 视频对比检测

对比转码视频和原片，智能判断转码是否引入新问题

python3 detect.py transcoded.mp4 --original original.mp4

3️⃣ Excel批量检测

从Excel批量检测转码视频

python3 detect.py data.xlsx

4️⃣ Excel批量对比检测 ✨ 新功能

从Excel批量对比检测，自动下载原片并与转码视频对比

python3 detect.py data.xlsx --compare --start 0 --stop 10

特点：

  • ✅ 自动使用bucket命令下载原片
  • ✅ 自动生成转码视频URL
  • ✅ 支持行号范围（start/stop）
  • ✅ 批量对比报告

🔍 检测类型

  • ✅ 黑屏（black_screen）- 画面完全或大部分为黑色
  • ✅ 绿屏（green_screen）- 绿幕或绿色异常
  • ✅ 马赛克/模糊（mosaic）- 画面模糊、马赛克
  • ✅ HDR异常（hdr_anomaly）- 过曝/欠曝
  • ✅ 视频卡顿（freeze）- 画面静止/卡顿
  • ✅ 解码错误（decode_error）- 视频解码失败
  • ⚠️ 偏色（color_cast）- 色彩失衡（默认关闭）
  • ⚠️ 人脸异常（face_anomaly）- 人脸模糊（默认关闭）

🚀 快速开始

安装依赖

pip3 install opencv-python numpy pandas openpyxl

基本使用

# 单视频检测
python3 detect.py video.mp4 -s 30

# 对比检测
python3 detect.py transcoded.mp4 --original original.mp4 -s 15

# Excel批量对比（前5个视频）
python3 detect.py data.xlsx --compare --start 0 --stop 5 -s 30

场景 命令 说明

🎯 参数说明

基本参数

-o, --output           # 输出目录（默认：./video_anomaly_output）
-t, --types            # 检测类型，逗号分隔
-s, --sample-interval  # 采样间隔（帧）
--threshold            # 异常阈值（0-1）

对比检测

--original             # 原片视频路径
--compare              # Excel批量对比模式
--start                # 起始行号（从0开始）
--stop                 # 结束行号（不包含）

批量检测

--limit                # 限制处理数量
--batch-output         # 批量报告路径

输出控制

--save-frames          # 保存检测帧
--json                 # JSON格式输出
--keep-downloaded      # 保留下载的视频

💡 使用建议

首次使用

# 1. 测试单视频
python3 detect.py test.mp4 -s 30

# 2. 测试对比检测
python3 detect.py transcoded.mp4 --original original.mp4 -s 30

# 3. 测试Excel批量对比（前3个）
python3 detect.py data.xlsx --compare --start 0 --stop 3 -s 30

生产环境

# 批量对比检测（分批处理）
python3 detect.py data.xlsx --compare --start 0 --stop 50 -s 15 \
  --batch-output batch1.json

python3 detect.py data.xlsx --compare --start 50 --stop 100 -s 15 \
  --batch-output batch2.json

夜间批处理

# 后台运行，处理所有视频
nohup python3 detect.py data.xlsx --compare -s 15 \
  --batch-output full_report.json > log.txt 2>&1 &

⚠️ 注意事项

Excel批量对比检测前置要求

  1. Bucket命令

    • 必须已安装bucket命令行工具
    • 必须配置PATH环境变量
    • 必须有访问video-def bucket的权限

    验证：

    bucket --help

  2. Excel文件格式

    • 必需列：photo_id
    • 必需列：blob_store_key

  3. 网络要求

    • 需要公司网络或VPN
    • 转码视频URL需要可访问

🆘 故障排除

问题1：bucket命令不存在

解决：安装bucket命令并配置PATH环境变量

问题2：原片下载失败

解决：检查photo_id是否正确，确认bucket中是否存在该文件

问题3：检测速度太慢

解决：使用 -s 30 或更大的采样间隔加快速度

问题4：Excel列名不匹配

解决：检查Excel列名是否完全一致，确认没有多余的空格

视频异常检测工具 v4.3.0

全自动视频质量检测工具，支持多种输入源和检测模式。

✨ 核心功能

输入支持

  • ✅ 本地视频文件 - 直接检测本地视频
  • ✅ 视频URL - 自动下载并检测（支持保留/清理）
  • ✅ Excel数据源 - 批量检测多个视频
  • ✅ 原片对比 - 判断转码是否引入新问题

检测模式

  1. 单视频检测 - 检测一个视频的所有异常
  2. 对比检测 - 对比转码视频和原片，识别转码引入的问题
  3. Excel批量检测 - 从Excel批量检测转码视频
  4. Excel批量对比 - 从Excel批量下载原片并对比检测

异常类型（8种）

类型 说明 检测条件 状态
black_screen 黑屏 持续≥3秒，亮度<20 ✅ 启用
green_screen 绿屏 持续≥3秒，绿色通道>1.5x ✅ 启用
mosaic 马赛克/模糊 持续≥2秒，清晰度<100 ✅ 启用
hdr_anomaly HDR异常 持续≥5秒，异常像素>30% ✅ 启用
freeze 视频卡顿 持续≥3秒，帧间MSE<10 ✅ 启用
decode_error 解码错误 帧读取失败 ✅ 启用
color_cast 偏色 RGB偏差>0.50 ⚠️ 默认关闭
face_anomaly 人脸异常 持续≥2秒，清晰度<100 ⚠️ 默认关闭

注意：所有检测自动忽略视频末尾3秒。

🚀 快速开始

1. 单视频检测

# 检测本地视频
python3 {baseDir}/scripts/detect.py /path/to/video.mp4

# 检测URL视频（自动下载）
python3 {baseDir}/scripts/detect.py https://example.com/video.mp4

# 快速检测（每30帧采样）
python3 {baseDir}/scripts/detect.py video.mp4 -s 30

2. 对比检测

# 对比转码视频和原片
python3 {baseDir}/scripts/detect.py transcoded.mp4 --original original.mp4

# 快速对比（推荐）
python3 {baseDir}/scripts/detect.py transcoded.mp4 --original original.mp4 -s 15

# 对比URL视频
python3 {baseDir}/scripts/detect.py "https://example.com/transcoded.mp4" \
  --original "https://example.com/original.mp4"

对比逻辑（v4.1.0+）：

  • 按异常类型分组对比
  • 转码有某类型，原片也有 → ✅ 不是新异常
  • 转码有某类型，原片没有 → ⚠️ 新异常
  • 解码错误特殊处理

3. Excel批量检测

# 批量检测转码视频
python3 {baseDir}/scripts/detect.py data.xlsx --limit 10

# 快速批量检测
python3 {baseDir}/scripts/detect.py data.xlsx --limit 10 -s 30

Excel要求：

  • 必需列：media_processing_detect_result.blob_store_key
  • 格式：JSON字符串，包含 key 字段

4. Excel批量对比（自动下载原片）

# Excel批量对比检测
python3 {baseDir}/scripts/detect.py data.xlsx --compare

# 测试前3个视频（推荐首次使用）
python3 {baseDir}/scripts/detect.py data.xlsx --compare --start 0 --stop 3 -s 30

# 批量处理指定范围
python3 {baseDir}/scripts/detect.py data.xlsx --compare --start 0 --stop 20 -s 15

# 保存批量对比报告
python3 {baseDir}/scripts/detect.py data.xlsx --compare --start 0 --stop 10 \
  --batch-output comparison_report.json

Excel要求：

  • 必需列1：media_processing_detect_result.photo_id（用于下载原片）
  • 必需列2：media_processing_detect_result.blob_store_key（生成转码URL）
  • 前置要求：需要安装并配置 bucket 命令

Bucket下载重试（v4.3.0）：

  • 命令成功但文件不存在时，等待5秒后重试
  • 最多重试2次（共3次尝试）
  • 提高下载成功率

🎯 参数说明

基本参数

-o, --output DIR              # 输出目录（默认：./video_anomaly_output）
-t, --types TYPES             # 检测类型（逗号分隔）
-s, --sample-interval N       # 采样间隔（帧）
--threshold FLOAT             # 异常阈值（0-1）

对比检测参数

--original PATH               # 原片视频路径（启用对比检测）

Excel批量检测参数

--limit N                     # 限制处理数量（用于调试）
--batch-output PATH           # 批量报告路径（JSON格式）

Excel批量对比参数

--compare                     # 启用Excel批量对比模式
--start N                     # 起始行号（从0开始）
--stop N                      # 结束行号（不包含）

输出控制

--save-frames                 # 保存检测帧
--keep-downloaded             # 保留下载的视频
--json                        # JSON格式输出

🔧 高级功能

OpenCV失败时的备用检测（v4.2.0+）

当OpenCV无法打开视频时，自动切换到ffmpeg_atlas检测：

支持的检测（v4.3.0+）：

  • ✅ 解码错误检测
  • ✅ 黑屏检测（持续≥3秒，使用blackdetect过滤器）
  • ✅ 冻结帧检测（持续≥3秒，使用freezedetect过滤器）

无法检测：

  • ❌ 绿屏、马赛克、HDR异常、人脸异常、偏色

自动切换示例：

⚠️  无法打开视频文件: video.mp4
切换到ffmpeg_atlas备用检测方案...

🔍 使用ffmpeg_atlas检测黑屏...
   ⚠️  发现黑屏: 5.5s-10.2s (持续4.7秒)

🔍 使用ffmpeg_atlas检测冻结帧...
   ⚠️  发现冻结帧: 22.4s-26.4s (持续4.0秒)

🔍 使用ffmpeg_atlas检测解码错误...
   ✅ 视频可以正常解码

对比检测逻辑（v4.1.0）

按异常类型分组对比：

# 规则1：解码错误特殊处理
转码有decode_error，原片无 → 新异常
转码有decode_error，原片也有 → 不是新异常

# 规则2：质量异常按类型判断
转码有某类型异常，原片也有该类型 → 不是新异常
转码有某类型异常，原片没有该类型 → 新异常

# 规则3：原片异常更多
原片异常类型比转码多 → 转码改善了质量

💡 使用技巧

1. 快速检测

# 每30帧采样，约1秒1次，速度快30倍
python3 scripts/detect.py video.mp4 -s 30

2. 关键检测

# 只检测最重要的问题
python3 scripts/detect.py video.mp4 -t freeze,decode_error

3. 分批处理

# Excel批量对比时分批处理，避免单次时间过长
python3 scripts/detect.py data.xlsx --compare --start 0 --stop 50 -s 15
python3 scripts/detect.py data.xlsx --compare --start 50 --stop 100 -s 15

4. 夜间批处理

# 后台运行
nohup python3 scripts/detect.py data.xlsx --compare -s 15 \
  --batch-output full_report.json > log.txt 2>&1 &
