视频异常检测工具

对视频文件进行全维度视觉异常检测，识别并定位视频中存在的各类视觉质量问题。

功能特性

  • 🎥 视频帧提取与分析
  • 🔍 多维度异常检测（花屏、绿屏、黑屏、马赛克、偏色，卡顿，解码错误等）
  • 📍 精确定位异常位置（时间段、区域坐标）
  • 📊 详细的检测报告和判断依据
  • 🖼️ 异常帧自动保存

基本用法

# 检测视频中的所有异常
python3 {baseDir}/scripts/detect.py /path/to/video.mp4

# 指定输出目录
python3 {baseDir}/scripts/detect.py /path/to/video.mp4 --output /path/to/output

# 仅检测特定类型的异常
python3 {baseDir}/scripts/detect.py /path/to/video.mp4 --types black_screen,green_screen,mosaic

# 设置采样间隔（每N帧检测一次，提高速度）
python3 {baseDir}/scripts/detect.py /path/to/video.mp4 --sample-interval 30

# 生成详细报告
python3 {baseDir}/scripts/detect.py /path/to/video.mp4 --detailed-report

检测类型

支持的异常检测类型：

类型 说明 检测依据
black_screen 黑屏检测 帧平均亮度低于阈值
green_screen 绿屏检测 绿色通道占比异常高
花屏 花屏检测 高频噪声、色块异常分布
mosaic 马赛克/模糊 图像清晰度、边缘检测
color_cast 偏色检测 RGB通道平衡分析
hdr_anomaly HDR异常 过曝/欠曝区域检测
freeze 视频卡顿 连续帧相似度检测
decode_error 解码错误 帧读取失败、帧跳过检测
face_anomaly 人脸异常（默认关闭） 人脸模糊、遮挡、光照异常

参数说明

  • --output, -o: 输出目录（默认：./video_anomaly_output）
  • --types, -t: 指定检测类型，逗号分隔（默认：全部）
  • --sample-interval, -s: 采样间隔，每N帧检测一次（默认：1，即每帧都检测）
  • --threshold: 异常判定阈值（默认：0.7）
  • --save-frames: 保存所有检测帧图片（默认不保存，节省磁盘空间）
  • --json: 输出JSON格式结果

环境要求

  • Python 3.7+
  • ffmpeg
  • opencv-python
  • numpy
  • pillow
  • scikit-image (可选，用于更精确的图像质量评估)

安装依赖

pip3 install opencv-python numpy pillow scikit-image
brew install ffmpeg  # macOS

高级用法

批量检测

# 检测目录中的所有视频
for video in /path/to/videos/*.mp4; do
  python3 {baseDir}/scripts/detect.py "$video" --output "./output/$(basename "$video" .mp4)"
done

自定义检测阈值

# 使用配置文件
python3 {baseDir}/scripts/detect.py /path/to/video.mp4 --config custom_config.json

配置文件示例：

{
  "black_screen": {
    "brightness_threshold": 15,
    "min_duration_frames": 3
  },
  "mosaic": {
    "sharpness_threshold": 25,
    "detection_area": "center"
  },
  "face_anomaly": {
    "enable_blur_detection": true,
    "blur_threshold": 100
  }
}

性能优化

  • 使用 --sample-interval 降低检测频率，提高速度
  • 使用 --types 仅检测需要的异常类型
  • 默认不保存帧图片，节省磁盘空间和IO时间（如需保存使用 --save-frames）
  • 多线程处理：脚本自动使用多核CPU

注意事项

  1. 视频文件较大时，完整检测可能耗时较长，建议使用采样间隔
  2. 某些异常检测（如人脸异常）需要额外的模型文件
  3. HDR视频可能需要特殊处理，建议先转换为SDR
  4. 默认不保存帧图片，如需保存请添加 --save-frames 参数

故障排除

Q: 提示缺少OpenCV模块

pip3 install opencv-python opencv-contrib-python

Q: ffmpeg未找到

# macOS
brew install ffmpeg

# Ubuntu/Debian
sudo apt-get install ffmpeg

Q: 检测速度太慢

  • 使用 --sample-interval 30 每秒检测1帧
  • 使用 --types 仅启用需要的检测类型
  • 确保SSD硬盘，避免机械硬盘IO瓶颈

🎯 功能特性

支持的异常检测类型

检测类型 说明 技术原理
黑屏 (black_screen) 检测画面完全或大部分为黑色 持续≥3秒，亮度<20，忽略视频末尾3秒
绿屏 (green_screen) 检测绿幕或绿色异常 绿色通道占比 > 1.5倍其他通道
马赛克/模糊 (mosaic) 检测画面模糊、马赛克 Laplacian清晰度 < 100
偏色 (color_cast) 检测色彩失衡 RGB通道偏差 > 0.15
HDR异常 (hdr_anomaly) 检测过曝/欠曝 持续≥5秒，异常像素>30%
视频卡顿 (freeze) 检测画面静止/卡顿 连续帧MSE < 10
解码错误 (decode_error) 检测视频解码失败 帧读取失败、帧跳过
人脸异常 (face_anomaly) ⚠️ 检测人脸模糊、异常（默认关闭） Haar级联 + 人脸清晰度 < 350

输出信息

每个检测到的异常包含：

  • ✅ 异常类型 - 具体是什么问题
  • ✅ 时间位置 - 精确到帧号和秒数
  • ✅ 时间段 - 连续异常会合并为时间段（如”5.0s - 10.2s”）
  • ✅ 空间位置 - 区域描述（如”顶部20%区域”、”中心区域(30%-70%)”）
  • ✅ 置信度 - 0-1之间，表示检测可靠性
  • ✅ 判断依据 - 详细的技术指标和阈值
  • ✅ 建议 - 人类可读的问题描述

🚀 快速开始

基本用法

# 进入skill目录
cd ~/.openclaw/workspace/skills/video-anomaly-detector-1.0.0

# 检测视频
python3 scripts/detect.py /path/to/video.mp4

# 使用快捷脚本（自动检查依赖）
bash scripts/detect.sh /path/to/video.mp4

常用参数

# 指定输出目录
python3 scripts/detect.py video.mp4 --output ./my_results

# 快速检测（每30帧检测一次，约1秒1次）
python3 scripts/detect.py video.mp4 --sample-interval 30

# 仅检测特定类型
python3 scripts/detect.py video.mp4 --types black_screen,green_screen,mosaic

# 调整检测灵敏度（阈值越低越敏感）
python3 scripts/detect.py video.mp4 --threshold 0.5

# 保存所有检测帧（默认不保存，节省空间）
python3 scripts/detect.py video.mp4 --save-frames

# 输出JSON格式
python3 scripts/detect.py video.mp4 --json

# 启用人脸异常检测（默认关闭）
python3 scripts/detect.py video.mp4 --types black_screen,green_screen,mosaic,color_cast,hdr_anomaly,face_anomaly

注意：

  • 默认不保存帧图片以节省磁盘空间，如需保存请添加 --save-frames 参数
  • face_anomaly（人脸异常检测）默认已关闭，如需使用请通过--types参数手动启用

💡 使用场景

1. 会议录像质量检查

python3 scripts/detect.py meeting.mp4 \
  --types black_screen,face_anomaly \
  --sample-interval 30 \
  --output ./meeting_check

2. 批量视频检测

#!/bin/bash
for video in /path/to/videos/*.mp4; do
  output_dir="./output/$(basename "$video" .mp4)"
  python3 scripts/detect.py "$video" \
    --output "$output_dir" \
    --sample-interval 30 \
    --save-frames
done

3. 直播录制质量监控

python3 scripts/detect.py live_stream.mp4 \
  --types black_screen,green_screen,hdr_anomaly \
  --sample-interval 60 \
  --threshold 0.8

⚙️ 性能优化

提速技巧

  1. 使用采样间隔

    --sample-interval 30  # 每30帧检测一次（1秒/次 for 30fps视频）

  2. 限制检测类型

    --types black_screen,green_screen  # 只检测最关键的异常

  3. 保存帧

    --save-frames  # 保存图片

内存优化

  • 脚本逐帧处理，内存占用稳定（约200-500MB）
  • 大视频建议使用SSD，避免IO瓶颈

处理大文件

# 2小时4K视频建议配置
python3 scripts/detect.py big_video.mp4 \
  --sample-interval 60 \
  --types black_screen,green_screen \
  --save-frames \
  --threshold 0.8

❓ 常见问题

Q1: 检测速度很慢怎么办？

A: 使用采样间隔参数

--sample-interval 30  # 每30帧检测一次

Q2: 误报太多怎么办？

A: 提高置信度阈值

--threshold 0.85  # 默认0.7，提高到0.85减少误报

Q3: 漏检问题怎么办？

A: 降低阈值或减少采样间隔

--threshold 0.5 --sample-interval 10

Q4: 磁盘空间不足？

A: 不保存帧图片

--no-save-frames

Q5: ImportError: No module named ‘cv2’

A: 安装依赖

pip3 install opencv-python numpy pillow

📈 技术细节

检测算法

检测类型 算法 复杂度
黑屏 灰度均值 O(n)
绿屏 RGB通道统计 O(n)
模糊 Laplacian方差 O(n log n)
偏色 RGB比例分析 O(n)
HDR 像素统计 O(n)
人脸 Haar Cascade O(n²)
